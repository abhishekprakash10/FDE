Edit Entry Drawer Admin Only
Administrators can click Edit on any entry row to open the Edit Entry drawer, which slides in from the right. All form fields are pre-populated with the existing record's data, allowing quick inline corrections. The score picker shows the current selection highlighted. Changes are persisted to the database on clicking Save Changes, after which the drawer closes and the Entries table refreshes automatically.

Delete Confirmation Drawer Admin Only
To prevent accidental data loss, clicking the Delete button opens a confirmation drawer rather than deleting the record immediately. The panel clearly identifies the record to be deleted by its ID and displays a permanent-action warning. The destructive Delete Entry button is styled in red to reinforce the severity of the action; clicking Cancel dismisses the drawer without making changes.

Tech Stack
PulseCheck is built as a decoupled full-stack application with a React frontend communicating with a Flask REST API backed by a SQLite database.
Layer Technology Version Purpose
Frontend React 18.x Component-based UI rendering, SPA routing
Frontend Vite 5.x Development server and production bundler
Frontend React Router DOM 6.x Client-side page routing and navigation
Frontend Tailwind CSS 3.x Utility-first styling, responsive layout
Frontend Axios 1.x HTTP client for REST API communication
Backend Python / Flask 3.11 / 3.x REST API server, request routing, JSON responses
Backend Flask-CORS 4.x Cross-Origin Resource Sharing headers for frontend
Backend Flask-SQLAlchemy 3.x ORM layer for database model definition and queries
Database SQLite 3.x Lightweight embedded relational database storage
Auth Session-based (in-memory) — Role differentiation: Viewer vs Administrator
Dev Tools ESLint + Prettier Latest Code linting and formatting consistency
Package Mgr npm / pip — Frontend and backend dependency management
API Reference
All API endpoints are served under http://localhost:5000/api. Viewer-accessible endpoints require no token; Admin endpoints require the session to have been established with the administrator password.
Method Path Auth Description
POST /api/auth/login Public Authenticate a user. Accepts { name, role, password? }. Returns session token and user role.
POST /api/auth/logout Any Invalidate the current user session and clear stored credentials.
GET /api/entries Any Retrieve all feedback entries. Supports optional query params: q (keyword), program, min_score.
GET /api/entries/:id Any Retrieve a single feedback entry by its numeric ID. Returns 404 if not found.
POST /api/entries Any Create a new feedback entry. Requires body { respondent, program, score, remarks? }.
PUT /api/entries/:id Admin Update an existing entry. Partial updates supported; only provided fields are modified.
DELETE /api/entries/:id Admin Permanently delete the specified entry. Returns { message: "Deleted" } on success.
GET /api/stats/overview Any Returns aggregate stats: total entries, average score, five-star count, unique program count.
GET /api/stats/by-program Any Returns per-program averages and entry counts, sorted by average score descending.
GET /api/programs Any Returns distinct list of program names for populating filter dropdowns on the Explore page.
Database Schema
PulseCheck uses a single-table SQLite schema. All data is stored in feedback.db located in the backend root directory. The schema can be initialized via flask db init or by running python app.py with the auto-create flag enabled.
TABLE feedback_entry
Column Type Constraints Description
id INTEGER PRIMARY KEY, AUTOINCREMENT Unique identifier for each feedback entry.
respondent VARCHAR(120) NOT NULL Full name of the person submitting the feedback.
program VARCHAR(200) NOT NULL Name of the training program or event being rated.
score INTEGER NOT NULL, CHECK (score BETWEEN 1 AND 5) Numeric rating from 1 (Terrible) to 5 (Excellent).
remarks TEXT NULLABLE Optional free-form comments or observations from the respondent.
created_at DATETIME NOT NULL, DEFAULT CURRENT_TIMESTAMP Timestamp of when the entry was submitted, auto-set by the server.
Sample INSERT
-- Insert a new feedback entry
INSERT INTO feedback_entry (respondent, program, score, remarks)
VALUES ('Priya Sharma', 'Python Basics', 5, 'Very well structured course.');
Sample SELECT (stats query)
-- Average score per program
SELECT program,
    ROUND(AVG(score), 1) AS avg_score,
    COUNT(*) AS entry_count
FROM feedback_entry
GROUP BY program
ORDER BY avg_score DESC;
Roles & Permissions
PulseCheck uses a two-role access model. Roles are established at sign-in time and stored in the browser session. No persistent accounts are maintained; roles are verified on every API call for admin-gated endpoints.
Permission Viewer Administrator
